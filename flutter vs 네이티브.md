# Flutter란 무엇인가? 네이티브와의 차이점 정리

**Flutter(플러터)**는 구글(Google)이 개발한 오픈소스 UI 소프트웨어 개발 도구

 하나로 안드로이드, iOS, 웹, 데스크톱 앱을 동시에 제작할 수 있는 
프로그램



## 1. Flutter vs 네이티브: 핵심 개념

* ```네이티브(Native):``` 각 운영체제 전용 언어(Android: Kotlin/Java, iOS: Swift)를 사용
* ```Flutter:``` 구글에서 개발한 dart를 사용





## 2. 주요 차이점 비교표

| 구분 | 네이티브  | 플러터 |
| :--- | :--- | :--- |
| **개발 언어** | Kotlin (안드로이드), Swift (iOS) | Dart |
| **코드베이스** | 플랫폼별 개별 작성 | 단일 코드베이스 (Single Source) |
| **성능** | 최상 | 우수(네이티브가 조금 더 좋음)  |
| **개발 속도** | 상대적으로 느림 | 매우 빠름 |
| **인력 구성** | 안드로이드/iOS 개발자 각각 필요 | Flutter 개발자 한 팀으로 가능 |

---

## 3. Flutter의 장단점

###  장점
- **생산성:** 한 번의 코딩으로 여러 플랫폼에 배포 가능하여 비용과 시간이 절감
- **UI 일관성:** 안드로이드와 아이폰에서 똑같은 디자인 가능


###  단점
- **앱 용량:** 엔진 용량이 포함되어 네이티브 앱보다 기본 설치 크기가 큼.
- **네이티브 종속성:** 카메라, 블루투스 등 복잡한 하드웨어 기능을 다룰 때 때때로 네이티브 코드를 직접 작성해야 할 수도 있음.
